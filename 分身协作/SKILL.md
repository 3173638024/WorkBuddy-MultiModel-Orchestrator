# 👥 分身协作 — Agent 多模型协作

> 一个你不够用？分身三个，各司其职，最后合体。

---

## ⚡ 一句话搞定

| 你说 | 效果 |
|------|------|
| **「分身，帮我开发一个博客系统」** | spawn 3 个 Agent：后端+前端+文档，并行写代码 |
| **「分身协作，分析这个项目并重构」** | 拆解→分配不同模型→各自写文件→审查整合 |

---

## 🎯 什么任务用它？

| ✅ 完美场景 | ❌ 别用它 |
|------------|---------|
| 开发含多个模块的项目（前后端+文档） | 纯文本分析、推理 |
| 同时生成多个独立文件 | 一句话能回答的问题 |
| 需要代码编辑+文件输出的复杂任务 | 需要快速出结果的场景 |

**本质：** spawn WorkBuddy Agent 来"动手干活"——写代码、生成文件、编辑项目。和前两个智囊团的不同在于：它们只能分析出报告，分身能真的写代码。

---

## ⚠️ 必须知道的事

**这是三个 Skill 中最慢的。** 因为 spawn Agent 需要 30 秒起步，完整协作 5-10 分钟。智囊团 60 秒能搞定的事，分身要 5 分钟。

**为什么还留着它？** 因为它是唯一能"写代码"的。当你需要同时生成多个代码文件时，前两个 API 编排 Skill 做不到——它们只能产出文本。

**建议：** 
- 要分析/推理 → 用 🧠 智囊团（快）
- 要写代码/生成文件 → 用 👥 分身协作（慢但能干活）
- 不会同时用两个——任务类型完全不同
